# CampusMast 系统架构设计

> 当前基线：本地测试 / 本地验收版
> 更新日期：2026-05-26

---

## 1. 总览

CampusMast 采用前后端分离架构，前端、后端都在本机运行；MySQL 和 Redis 通过根目录 `docker-compose.yml` 在本机启动。

```text
浏览器
  -> Vue 3 / Vite 前端
  -> FastAPI HTTP API / WebSocket
  -> MySQL（业务数据）
  -> Redis（实时推送、未读、缓存）
  -> 后端本地 uploads 目录（任务图片）
  -> DeepSeek API（可选，内容审核）
```

本项目不再维护生产上线部署，不使用 Alembic 迁移。数据库结构由 SQLAlchemy 模型直接建表，初始化和演示数据通过 `python -m app.init_data`、`python -m app.demo_data` 生成。

## 2. 技术选型

| 层 | 技术 | 用途 |
|---|---|---|
| 前端 | Vue 3、TypeScript、Vite、Pinia、Vue Router、Axios | 页面、状态、路由、接口调用 |
| 地图 | Leaflet、OpenStreetMap | OSM 底图、楼宇覆盖层、任务点位 |
| 后端 | FastAPI、SQLAlchemy 2.0、Pydantic | HTTP API、WebSocket、模型校验 |
| 数据 | MySQL 8.0 | 用户、任务、钱包、信用、聊天、审核、配置等业务数据 |
| 实时与缓存 | Redis | WebSocket 发布订阅、未读数、推荐缓存 |
| 上传 | FastAPI 本地文件存储 | 任务图片保存到 `backend/uploads/` |
| 审核 | 敏感词预检 + DeepSeek API + 人工复审 | 发布前内容审核；未配置 Key 时按 refuse/MEDIUM 进入复审 |
| 测试 | pytest、vue-tsc、Vite build | 后端回归与前端构建验证 |

## 3. 后端分层

```text
Router / WebSocket Gateway
  -> Dependencies（登录用户、管理员、数据库会话）
  -> Service（任务、钱包、信用、推荐、审核、聊天、后台）
  -> SQLAlchemy Models
  -> MySQL / Redis / uploads / DeepSeek
```

| 模块 | 说明 |
|---|---|
| `auth` | 注册、登录、刷新登录、改密码、个人资料、公开主页 |
| `task` | 发布、接单、提交证明、确认、拒绝、争议、取消、放弃、评价 |
| `wallet` | 余额、流水、充值、提现、冻结、结算、争议分账 |
| `credit` | 信用分、评价、信用统计、权重配置 |
| `notification` | 通知列表、已读、全部已读、实时推送 |
| `chat` | 任务聊天会话、消息、已读、HTTP 兜底 |
| `map` | OSM 楼宇、当前用户选点解析、附近任务、距离计算 |
| `recommendation` | 智能推荐、推荐解释与推荐快照 |
| `moderation` | 敏感词预检、DeepSeek 初审、人工复审、用户审核记录 |
| `admin` | 用户管理、任务管理、争议处理、审核记录、系统配置、后台统计 |
| `upload` | 本地任务图片上传 |

## 4. 关键链路

### 4.1 任务发布

```text
前端发布任务
  -> 本地上传图片并拿到 fileUrl
  -> FastAPI 校验任务参数
  -> 根据 buildingCode 或地图坐标确定楼宇
  -> 本地敏感词预检；命中则拒发并留审核记录
  -> DeepSeek 初审：accept 直接发布 / reject 拒发 / refuse 发布后待人工复审
  -> DeepSeek 未配置或异常时按 refuse/MEDIUM 进入人工复审
  -> 钱包冻结赏金
  -> 创建任务、通知、审核记录
  -> 返回任务详情
```

### 4.2 任务完成与结算

```text
接单人提交完成证明
  -> 发布者确认完成
  -> 钱包从冻结金额结算给接单人
  -> 写入流水和任务日志
  -> 更新信用与通知
```

### 4.3 聊天与通知

```text
HTTP 操作产生业务事件
  -> 写入通知 / 聊天消息
  -> Redis 发布事件
  -> WebSocket 推送到在线用户
  -> 前端未在线时仍可通过 HTTP 列表读取
```

### 4.4 地图与楼宇

```text
初始化脚本写入 OSM 楼宇数据
  -> /api/map/buildings 返回楼宇点位和多边形
  -> /api/map/current-location 根据用户选点或默认楼宇解析当前位置
  -> 前端 Leaflet 渲染 OSM 底图和楼宇覆盖层
  -> 发布任务可从地图选点
  -> /api/tasks 支持 userLatitude/userLongitude 或 nearBuildingCode 作为距离排序原点
  -> 任务列表返回 distanceMeters，用于列表和地图任务卡展示具体距离
```

### 4.5 推荐与审核可观测性

```text
任务大厅 sortBy=recommended / 推荐接口
  -> 读取用户画像：偏好分类、默认楼宇、活跃时段、履约成功率
  -> 计算类别、距离、成功率、活跃时间四项得分
  -> 返回 recommendation.reason 与 signals，便于前端展示和测试验证

管理员复审 refuse 任务
  -> approve：清除 needsAdminReview，保留任务上架
  -> reject：PENDING 任务置为 CLOSED_BY_ADMIN，解冻赏金，写 task_logs
  -> 用户侧 /api/moderation/my-records 可查看 AI 结果、风险提示、人工结论和理由
```

## 5. 本地运行基线

| 项 | 命令 / 说明 |
|---|---|
| 基础设施 | 根目录执行 `docker compose up -d`，只启动 MySQL 和 Redis |
| 后端依赖 | `cd backend && pip install -r requirements.txt` |
| 干净初始化 | `python -m app.init_data` |
| 演示数据 | `python -m app.demo_data` |
| 后端服务 | `uvicorn app.main:app --reload --port 9000` |
| 前端依赖 | `cd frontend && npm install` |
| 前端服务 | `npm run dev` |
| 后端测试 | `python -m pytest backend/tests` |
| 前端验证 | `npm run build` |

## 6. 配置项

| 环境变量 | 说明 |
|---|---|
| `DATABASE_URL` | 本地 MySQL 连接串 |
| `REDIS_URL` | 本地 Redis 连接串 |
| `JWT_SECRET` | 本地 JWT 密钥 |
| `LOCAL_UPLOAD_DIR` | 本地上传目录，默认 `uploads` |
| `LOCAL_UPLOAD_BASE_URL` | 上传文件访问前缀，默认 `/uploads` |
| `LOCAL_UPLOAD_MAX_SIZE_MB` | 单张图片大小限制，默认 5MB |
| `DEEPSEEK_API_KEY` | DeepSeek API Key；不填时走关键词兜底 |
| `DEEPSEEK_BASE_URL` | DeepSeek API 地址 |
| `CORS_ORIGINS` | 本地前端地址白名单 |

## 7. 约束

- `buildingCode` 继续作为任务地点和 OSM 楼宇之间的稳定内部编码。
- `backend/uploads/` 是本地运行数据，不提交到仓库。
- 生产部署、镜像构建和数据库迁移不属于当前基线。
- 历史评审记录保留原文；当前有效口径以 README、API 规范、数据库设计和本文档为准。
