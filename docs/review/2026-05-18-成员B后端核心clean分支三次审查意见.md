## 2. 当前阻塞问题

### 2.1 `credit.py` 仍引用已删除的 `build_stub_rating`

运行测试命令：

```bash
env PYTHONPATH=backend .venv312/bin/python -m pytest backend/tests
```

结果在 collection 阶段失败：

```text
ImportError while loading conftest '/private/tmp/campusmast-b-review/backend/tests/conftest.py'.
backend/tests/conftest.py:21: in <module>
    from app.main import app
backend/app/main.py:14: in <module>
    from app.routers import admin, auth, chat, credit, map, moderation, notification, recommendation, task, upload, wallet
backend/app/routers/credit.py:10: in <module>
    from app.services.credit_service import build_stub_rating
E   ImportError: cannot import name 'build_stub_rating' from 'app.services.credit_service'
```

问题文件：

```text
backend/app/routers/credit.py
```

当前仍是旧占位实现：

```python
from app.services.credit_service import build_stub_rating
from app.utils.response import success

router = APIRouter(prefix="/credit", tags=["Credit"])


@router.post("/ratings")
def create_rating() -> dict:
    return success(build_stub_rating())
```

但这一版 `backend/app/services/credit_service.py` 已经没有 `build_stub_rating`，真实互评逻辑也已经迁移到：

```text
POST /api/tasks/{task_id}/rating
```

对应实现位于：

```text
backend/app/routers/task.py
backend/app/services/credit_service.py
```

因此 `backend/app/routers/credit.py` 不能继续保留旧的 `/credit/ratings` 占位接口。

整改要求二选一：

#### 方案 A：保留空 router

如果本阶段不单独提供 `/api/credit/*` 接口，只保留任务评价入口，则将 `backend/app/routers/credit.py` 改为：

```python
from __future__ import annotations

from fastapi import APIRouter

router = APIRouter(prefix="/credit", tags=["Credit"])
```

这是最小修复方式。上一版 `origin/feat/b-backend-core` 中曾经就是这个方向。

#### 方案 B：实现真实信用查询接口

如果希望保留 Credit 模块接口，可以新增只读接口，例如：

```python
@router.get("/me")
def get_my_credit_profile(
    db: Session = Depends(get_db),
    current_user: User = Depends(get_current_user),
) -> dict:
    return success(get_credit_profile(db, current_user.id))
```

但这不是冻结 API 中明确要求的接口，若要新增 `/api/credit/me`，建议先征求组长确认，避免前端和 API 文档额外扩张。

推荐采用方案 A：先删除旧占位接口，保证应用能启动。
