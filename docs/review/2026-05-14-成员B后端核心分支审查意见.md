## 3. 业务实现需要补齐的具体问题

### 3.1 创建任务没有接入同步审核

冻结 PRD 要求：

- 发布任务前同步调用 DeepSeek 审核标题、描述和图片说明
- 高危内容直接拦截，不允许创建任务
- 低危内容允许提交，但标记为待管理员复审
- 审核结果写入审核记录表

但当前 `backend/app/services/task_service.py` 的 `create_task` 中直接写死：

```python
moderation_result=ModerationResult.ALLOW
needs_admin_review=False
```

并没有调用 `moderation_service`，也没有处理 `REVIEW` / `BLOCK`。

整改要求：

1. 在 `create_task` 中调用审核服务。
2. 审核结果为 `BLOCK` 时回滚，不创建任务、不冻结资金，返回 `MODERATION_BLOCKED`。
3. 审核结果为 `REVIEW` 时允许创建任务，但写入：

```python
moderation_result=ModerationResult.REVIEW
needs_admin_review=True
```

4. 审核结果为 `ALLOW` 时正常创建。
5. 审核记录必须落库，并能被后台审核列表读取。
6. 增加至少三类测试：

```text
test_create_task_blocks_high_risk_content_without_freezing_wallet
test_create_task_review_content_marks_needs_admin_review
test_create_task_allow_content_freezes_reward_and_writes_moderation_record
```

### 3.2 普通确认完成后没有即时刷新信用分

PRD 状态转换规则要求：

```text
PENDING_REVIEW -> COMPLETED：结算资金、更新信用、允许互评
```

当前 `confirm_task` 中完成了结算和状态流转，但没有调用 `_recalculate_participant_credit`。这会导致用户完成任务后，`requesterCreditScore`、`helperCreditScore`、`overallCreditScore` 仍保持旧值，直到其他地方手动重算。

整改要求：

在 `confirm_task` 完成状态流转后、事务提交前调用：

```python
_recalculate_participant_credit(db, task)
```

并增加测试：

```text
test_confirm_task_recalculates_requester_and_helper_credit
```

### 3.3 Helper 放弃任务后没有即时刷新信用分

PRD 状态转换规则要求：

```text
IN_PROGRESS -> PENDING：清空 helperId、更新信用、发通知
```

当前 `abandon_task` 清空了 `helper_id` 并回到 `PENDING`，但没有刷新放弃者信用。更严重的是，当前代码先执行：

```python
task.helper_id = None
```

再做状态转换。后续如果按 `task.helper_id` 重算，会丢失原 helper 信息。

整改要求：

1. 在清空 `helper_id` 之前保存原 helper：

```python
old_helper_id = task.helper_id
```

2. 状态流转和日志写入完成后，对 `old_helper_id` 重算信用。
3. 增加测试：

```text
test_abandon_task_recalculates_original_helper_credit
```

### 3.4 接单门槛配置键与当前 check 基线不一致

当前 `check` 的 bootstrap 默认配置键是：

```text
helperCreditThreshold
```

B 分支读取的是：

```text
task.acceptance.helperCreditThreshold
```

这会导致后台配置更新后，接单门槛逻辑仍然读取不到配置，继续使用默认值 `60.00`。

整改要求二选一：

1. B 的代码改为读取 `helperCreditThreshold`。
2. 或者由组长确认统一改配置键，并同步更新：
   - `backend/app/bootstrap.py`
   - `backend/app/services/config_service.py`
   - `docs/API接口规范.yaml`
   - `docs/数据库设计.md`
   - 相关测试

在没有 RFC 和组长确认前，建议采用第 1 种，避免扩大影响面。
