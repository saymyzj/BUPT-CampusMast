# 2026-05-25 C-owner 收口说明

## 1. 接口契约口径调整

由于本阶段需求与联调细节变动频繁，`docs/API接口规范.yaml` 暂不再表述为已经完全冻结的最终契约。

当前口径调整为：

- 开发期：以实际代码实现为主，OpenAPI 文档同步校准。
- 验收前：由组长统一核对代码、页面和接口，再冻结最终 OpenAPI 契约。
- 仍需组长拍板的内容：字段名、状态机、错误码、WebSocket 事件、信用字段、审核结果枚举。

## 2. WebSocket 收口

WebSocket 后端补齐以下能力：

- 频道鉴权：`notification:{userId}` 只能本人订阅；`chat:{taskId}` / `presence:{taskId}` 只能任务聊天参与者订阅。
- Redis 发布/订阅：本地进程先推送本进程连接，再通过 Redis 广播到其他进程。
- Redis 不可用时降级：本地演示仍可用，不阻塞单进程联调。
- 管理员可查看连接快照：`GET /api/admin/ws/connections`。
- 前端导航未读角标接入 `notification:{userId}`，支持断线自动重连。

## 3. 后台配置能力收口

后台配置从“接口存在”推进到“后台可操作”：

- `GET /api/admin/configs` 查询系统配置。
- `PUT /api/admin/configs/{key}` 支持新增和更新配置项。
- 后台“运营规则配置”可编辑配置 JSON。
- 后台“首页内容配置”可编辑首页公告、Banner、推荐位，并支持启停。

当前内置配置包括：

- `helperCreditThreshold`
- `credit.weights`
- `recommendation.weights`
- `moderation.mode`
- `homepageBannerRotationSeconds`

## 4. AI 审核最终口径

最终口径调整为：**敏感词预检 + DeepSeek 初审 + 人工复审**。

具体策略：

- 发布任务先执行本地敏感词预检；命中时直接 `BLOCK`，不创建任务、不冻结资金，但写审核记录。
- 未命中敏感词时调用 DeepSeek，对任务标题、描述与图片 URL 摘要进行同步审核。
- DeepSeek 返回 `accept` 时映射为 `ALLOW` 并直接发布；返回 `reject` 时映射为 `BLOCK` 并拒发；返回 `refuse` 时映射为 `REVIEW`，允许发布但进入人工复审。
- `refuse` 需要保留 `LOW` / `MEDIUM` / `HIGH` 风险提示；DeepSeek 未配置、超时或异常时按 `refuse` + `MEDIUM` 兜底。
- 人工复审 `approve` 会清除任务待复审标记；`reject` 会将仍处于 `PENDING` 的任务下架为 `CLOSED_BY_ADMIN`，解冻赏金并写任务日志。
- 审核记录统一写入 `moderation_records`，用户侧和后台均可查看 AI 结果、风险提示、人工结论和理由。

## 5. 联调演示数据脚本

新增脚本：

```bash
cd backend
source .venv/bin/activate
python -m app.demo_data
```

该脚本会清空业务表并生成演示数据：

- 管理员：`admin / 12345`
- 普通用户：`user01-user10 / 12345`
- 演示接单人：`user02 / 12345`
- 多状态任务：待接单、进行中、待验收、已完成、争议中
- 钱包余额、通知、聊天会话、审核记录、评价、系统配置、首页模块

一键初始化脚本：

```bash
python -m app.init_data
```

`init_data` 会清空并重建数据库，只保留 `admin` 与 `user01-user10`，钱包余额均为 `200.00`，适合干净测试；`demo_data` 适合联调演示。
